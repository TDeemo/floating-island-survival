# 漂浮大陆：1个月内可玩原型规划

## 1. 概述

本文档旨在拆解《漂浮大陆》游戏在1个月内可交付的核心功能清单，并创建对应的开发任务。基于游戏设计文档（`docs/game_design_document.md`）与技术架构方案（`docs/technical_architecture.md`），本规划聚焦于实现最小可行产品（MVP）——即能够完整呈现“探险-收集-建造”核心循环的可玩原型。

**目标**：在4周内交付一个包含基础岛屿生成、角色移动、资源采集、简易建造、昼夜循环、武器系统、种植原型与光照效果的可运行PC版本。

**核心原则**：
- 功能优先级：先实现核心循环，再丰富细节
- 技术可行性：充分利用现有Unity资产，避免过度复杂化
- 可测试性：每周产出可运行的构建版本

## 2. 核心功能清单

### 2.1 岛屿生成系统（核心）
- **基础地形生成**：基于Perlin噪声生成64×64网格地形，区分草地、森林、山脉、水域
- **岛屿类型差异化**：至少实现3种岛屿类型（森林、矿山、沼泽）的生态差异化
- **资源分布**：随机放置资源节点（树木、矿石、浆果丛）
- **港口生成**：每座岛屿边缘固定生成港口区域，包含船只交互点
- **性能优化**：分块加载机制，确保流畅体验

### 2.2 基础探险机制
- **角色移动控制**：WASD键盘控制，2D物理移动
- **相机跟随**：平滑跟随角色，边界约束
- **基础敌人AI**：巡逻、追击、攻击状态机（复用现有`Enemy.cs`）
- **碰撞与交互**：角色与地形、资源、敌人的基础碰撞检测
- **昼夜时间系统**：游戏内24分钟对应1天，可视化时间显示

### 2.3 资源收集系统
- **工具系统**：斧头（伐木）、镐（采矿）、手（采集）的基础交互
- **资源节点**：树木、矿石、浆果丛的可采集状态与动画
- **背包管理**：基础物品格子系统，重量/容量限制
- **采集反馈**：视觉/音效提示，资源进入背包

### 2.4 简易建造系统
- **建造模式**：快捷键进入，显示可放置网格
- **建筑清单**：工作台、熔炉、仓库、农田的基础建造
- **资源消耗**：建造时扣除对应资源
- **建筑功能**：工作台（打开制作界面）、仓库（存储资源）

### 2.5 基础UI系统
- **HUD元素**：生命值、耐力值、时间显示、背包快捷栏
- **背包界面**：格子式物品显示，拖拽/使用交互
- **建造界面**：建筑选择面板，资源需求显示
- **制作界面**：配方列表，材料检查，制作按钮
- **设置菜单**：基础音量、画面设置

### 2.6 测试场景
- **主岛场景**：玩家出生点，基础地形，初始工作台
- **探险岛屿场景**：程序生成的测试岛屿，包含各类资源与敌人
- **场景切换**：主岛←→探险岛屿的加载与过渡

### 2.7 种植系统（原型阶段）
- **农田建筑**：可放置的农田地块，基础土壤视觉效果
- **种植交互**：靠近农田打开种植界面，选择种子种植
- **生长可视化**：作物生长阶段（种子→幼苗→成熟）的精灵变化
- **收获机制**：成熟作物点击收获，获得农产品

### 2.8 光照系统（原型阶段）
- **手持火把**：制作配方（木材+纤维），装备后提供动态光源
- **2D光照效果**：使用URP 2D Light系统，圆形光照范围
- **黑暗视野**：光源范围外区域变暗/不可见
- **基础性能**：确保移动端兼容性的简化版本

## 3. 开发时间规划（周粒度）

### 第1周：基础框架与岛屿生成（3月6日-3月12日）

**目标**：搭建项目骨架，实现可探索的程序生成岛屿。

**核心任务**：
1. **项目初始化**（1天）
   - 创建Unity 2022.3 LTS项目，导入现有Assets
   - 配置URP 2D渲染管线，设置Pixel Perfect Camera
   - 建立Git仓库与基础文件夹结构

2. **岛屿生成系统**（3天）
   - 实现Perlin噪声地形生成算法
   - 创建Tilemap系统，支持草地、森林、山脉、水域图块
   - 实现资源节点随机分布算法（树木、矿石、浆果丛）
   - 添加港口生成逻辑（岛屿边缘固定位置）

3. **角色与相机**（2天）
   - 实现角色移动控制器（WASD + 物理）
   - 创建相机跟随脚本，适配岛屿边界
   - 导入基础角色精灵，设置动画状态机

4. **测试场景搭建**（1天）
   - 创建主岛场景（简单地形，初始建筑）
   - 创建探险岛屿生成测试场景
   - 实现基础场景管理（场景切换原型）

**交付物**：可运行构建，角色可在程序生成的岛屿上移动探索。

### 第2周：探险与资源收集（3月13日-3月19日）

**目标**：实现资源采集核心循环，添加基础敌人与UI。

**核心任务**：
1. **资源收集系统**（2天）
   - 实现工具交互系统（斧头伐木、镐采矿、手采集）
   - 创建资源节点脚本（生命值、采集时间、掉落物）
   - 添加采集动画与音效反馈

2. **背包与物品系统**（2天）
   - 实现背包数据模型（物品ID、数量、堆叠限制）
   - 创建背包UI界面（格子布局、物品图标）
   - 添加物品拖拽、使用、丢弃功能

3. **敌人AI与战斗**（2天）
   - 完善现有`Enemy.cs`脚本（攻击逻辑、伤害计算）
   - 实现角色攻击系统（近战攻击动画、伤害区域）
   - 添加生命值系统（玩家与敌人的受伤/死亡逻辑）

4. **昼夜时间系统**（1天）
   - 实现游戏内时间管理器（24分钟现实时间 = 1游戏日）
   - 添加时间可视化（HUD时钟、天空色渐变）
   - 创建时间阶段检测（白天、黄昏、夜晚）

**交付物**：可收集资源、与敌人战斗、查看背包的基本玩法循环。

### 第3周：建造与武器系统（3月20日-3月26日）

**目标**：实现基地建造与武器系统，完善游戏经济循环。

**核心任务**：
1. **建造系统**（2天）
   - 实现建造模式（网格显示、建筑预览）
   - 创建建筑数据模型（类型、资源需求、功能）
   - 添加建筑放置、旋转、拆除逻辑
   - 实现建筑功能接口（工作台、熔炉、仓库）

2. **武器系统**（2天）
   - 创建武器数据模型（伤害、攻速、特效、耐久度）
   - 实现武器切换系统（主/副武器槽）
   - 添加武器攻击效果（近战挥砍、远程射击）
   - 制作基础武器配方（木剑、石斧、短弓）

3. **UI系统完善**（2天）
   - 实现建造界面（建筑选择面板、资源检查）
   - 创建制作界面（配方列表、材料消耗、制作按钮）
   - 添加武器面板（当前装备显示、耐久度条）
   - 优化HUD布局与响应式设计

4. **港口撤离机制**（1天）
   - 实现港口交互系统（靠近船只触发撤离）
   - 添加撤离逻辑（返回主岛、保留收集资源）
   - 创建时间压迫感（夜晚倒计时提示）

**交付物**：可建造建筑、制作武器、完成探险撤离的完整核心循环。

### 第4周：种植、光照与整合（3月27日-4月2日）

**目标**：实现种植与光照系统原型，进行最终整合与优化。

**核心任务**：
1. **种植系统原型**（2天）
   - 实现农田建筑（可放置、土壤视觉效果）
   - 创建种植交互界面（种子选择、种植确认）
   - 添加作物生长逻辑（时间进度、阶段变化）
   - 实现收获机制（点击成熟作物获得农产品）

2. **光照系统原型**（2天）
   - 实现手持火把系统（制作配方、装备状态）
   - 集成URP 2D Light（点光源跟随、范围控制）
   - 添加黑暗视野效果（光源外区域变暗）
   - 优化光照性能（光源数量限制、视锥剔除）

3. **系统整合与平衡**（2天）
   - 连接所有系统接口（资源流、UI交互、数据同步）
   - 调整数值平衡（资源产出/消耗、敌人难度、时间节奏）
   - 修复关键Bug与性能问题
   - 添加基础音效与视觉反馈

4. **最终构建与测试**（1天）
   - 创建PC可执行构建版本
   - 进行基础功能测试（核心循环完整性）
   - 准备测试文档与反馈收集机制

**交付物**：包含全部8个核心系统的可玩原型，可进行内部测试与演示。

## 5. 风险与应对

| 风险 | 可能性 | 影响 | 应对策略 |
|------|--------|------|----------|
| 岛屿生成性能问题 | 中 | 高 | 提前实现分块加载，优化噪声算法；准备简化版生成方案 |
| 多系统集成复杂性 | 高 | 中 | 采用模块化设计，定义清晰接口；每周进行集成测试 |
| 美术资源延迟 | 中 | 中 | 使用占位符开发，美术资源异步集成；优先保证功能完整 |
| 网络同步延迟 | 低 | 高 | 原型阶段简化同步逻辑，使用本地模拟；标记为后续优化重点 |
| 时间估算不足 | 中 | 高 | 保持功能优先级，必要时裁剪次要功能；确保核心循环可运行 |

**关键风险点**：
- **第2周末**：如果资源收集系统延迟，将影响后续建造系统开发
- **第3周中**：武器系统与建造系统的接口集成需要重点测试
- **第4周初**：种植与光照系统作为新增功能，需严格控制范围

## 6. 验收标准

### 6.1 功能完整性
1. **核心玩法循环**：玩家可以完成"选择岛屿→探险采集→返回主岛→建造升级"的完整循环
2. **基础系统**：8个核心系统（岛屿生成、探险、资源收集、建造、UI、测试场景、种植原型、光照原型）均实现基本功能
3. **可玩性**：原型有明确的游戏目标（收集资源、建造建筑、生存过夜），能提供30分钟以上的游戏体验

### 6.2 技术可行性
1. **性能标准**：在目标PC配置（i5+GTX 1060）上保持稳定30FPS以上
2. **稳定性**：无崩溃性Bug，核心功能流程无阻断性错误
3. **扩展性**：系统架构符合技术方案设计，支持后续功能扩展

### 6.3 用户体验
1. **操作流畅**：角色移动、采集、建造等基础操作响应及时
2. **界面清晰**：基础UI元素（生命值、背包、时间）清晰可辨
3. **反馈明确**：关键操作（采集成功、受到伤害、建造完成）有视觉/音效反馈

### 6.4 文档完整性
1. **规划文档**：本`prototype_plan.md`文档包含详细的时间规划、资源依赖与风险应对
2. **测试指南**：第4周产出基础测试用例清单
3. **构建说明**：提供可执行文件的运行环境要求

## 7. 下一步行动

1. **立即开始**：按照第1周计划启动Unity项目初始化
2. **资源准备**：技术美术开始制作高优先级美术占位符
3. **团队分工**：根据人力资源表分配具体任务
4. **每周评审**：每周末检查进度，调整下周计划

**成功标志**：4周后产出可独立运行的PC原型，核心玩法循环完整，可进行内部测试与演示。
