# 基础建造系统测试指南

## 1. 测试环境设置

### 1.1 场景准备
1. 打开Unity项目，创建一个新的测试场景或使用现有主岛场景
2. 确保场景包含以下GameObject：
   - **GameManager**: 管理游戏状态和资源
   - **Player**: 带有PlayerResourceCollector脚本的角色
   - **Camera**: 相机跟随脚本
   - **资源节点**: 至少一个Wood和Stone资源节点用于收集测试

### 1.2 系统初始化
1. 创建空GameObject，命名为"BuildingSystem"
2. 添加以下组件：
   - **BuildingManager** (BuildingSystem.BuildingManager)
   - **BuildingSystemTest** (BuildingSystem.BuildingSystemTest)

3. 配置BuildingManager引用：
   - Workbench Prefab: 留空（系统会动态创建）
   - Farmland Prefab: 留空（系统会动态创建）
   - Campfire Prefab: 留空（系统会动态创建）
   - Buildable Layer Mask: 选择可建造的地形层

## 2. 功能测试流程

### 2.1 资源收集测试
1. 运行游戏，控制玩家移动到资源节点附近
2. 按E键收集资源：
   - 收集木材：游戏控制台应显示"Collected wood. Total: X"
   - 收集石头：游戏控制台应显示"Collected stone. Total: X"
3. 确认GameManager中的资源计数正确更新

### 2.2 建造模式基础测试
1. 按B键进入建造模式
   - 游戏控制台应显示"Entered Build Mode"
   - 屏幕上应出现建筑预览（绿色方块）
2. 移动鼠标查看预览跟随
3. 按数字键切换建筑类型：
   - 按1：切换到工作台（预览颜色变化）
   - 按2：切换到农田（预览颜色变化）
   - 按3：切换到篝火（预览颜色变化）
4. 按ESC键退出建造模式

### 2.3 建筑放置测试
1. 确保玩家有足够资源：
   - 工作台：10木材
   - 农田：5木材 + 3石头
   - 篝火：8木材 + 2石头
   
2. 进入建造模式（B键），选择工作台（按1）
3. 移动预览到空旷位置（显示绿色）
4. 左键点击放置建筑：
   - 控制台应显示"Workbench placed at (位置)"
   - 游戏资源应扣除相应数量
   - 放置的建筑应变为实体（非预览状态）

5. 重复测试农田和篝火

### 2.4 建造验证测试
1. 尝试在不可放置位置（如其他建筑上）放置建筑：
   - 预览应显示红色
   - 左键点击不应放置建筑
   - 资源不应被扣除

2. 尝试在资源不足时放置建筑：
   - 预览应为绿色但无法放置
   - 控制台应有提示信息

### 2.5 农田种植交互测试
1. 放置一个农田建筑
2. 确保玩家有至少1个药草资源（用于种子）
3. 靠近农田，按E键交互：
   - 第一次按E：应种植种子，控制台显示"Planted herb_seed..."
   - 农田视觉状态应从"空"变为"已种植"
   
4. 等待生长时间（默认120秒）：
   - 可加速测试：修改FarmlandBuilding的growthTime为10秒
   - 观察控制台生长进度日志（每10%进度）
   
5. 作物成熟后：
   - 农田视觉状态应从"已种植"变为"可收获"
   - 控制台显示"Crops are ready to harvest!"
   
6. 靠近农田，按E键收获：
   - 应收获2个药草资源
   - 控制台显示"Harvested 2 herbs! Total herbs: X"
   - 农田重置为空状态

### 2.6 篝火功能测试
1. 放置一个篝火建筑
2. 确认篝火在夜间自动点亮：
   - 修改GameManager的dayLengthInSeconds为60秒加速昼夜循环
   - 观察进入夜晚时篝火是否自动点亮
   
3. 测试光照效果：
   - 玩家进入篝火光照范围，控制台显示"Player entered campfire light area"
   - 玩家离开范围，控制台显示"Player left campfire light area"

4. 测试交互：靠近篝火按E键熄灭/点燃

### 2.7 数据持久化测试
1. 放置几个不同建筑
2. 保存游戏：
   - 控制台显示"Building data saved to: (路径)"
   - 检查保存文件是否存在
   
3. 重启游戏或重新加载场景
4. 建筑应自动从保存文件加载：
   - 控制台显示"Loaded X buildings from save file"
   - 现场应出现之前放置的建筑

## 3. 自动化测试

### 3.1 运行测试套件
1. 确保BuildingSystemTest组件的"Run Tests On Start"勾选
2. 运行游戏，观察自动测试执行
3. 在游戏视图的GUI中可手动点击"Run All Tests"按钮

### 3.2 测试结果解读
- **Test 1**: 管理器引用检查（应通过）
- **Test 2**: 资源扣除测试（应通过）
- **Test 3**: 建造模式切换测试（应通过）
- **Test 4**: 建筑放置测试（简化测试，应通过）
- **Test 5**: 农田交互测试（简化测试，应通过）
- **Test 6**: 持久化测试（应通过）

所有6项测试通过表示建造系统核心功能正常。

## 4. 常见问题排查

### 4.1 编译错误
如果出现编译错误：
1. 检查所有.cs文件的命名空间一致性
2. 确保没有缺少的Unity组件引用
3. 验证Light2D命名空间（可能需要调整Unity版本）

### 4.2 预制体引用问题
如果建筑无法实例化：
1. 确认BuildingManager的预制体字段不为空
2. 或允许系统动态创建预制体（已实现）

### 4.3 资源扣除异常
如果资源扣除不正确：
1. 检查GameManager单例是否正常初始化
2. 验证AddResource方法的正负数处理

### 4.4 保存/加载失败
如果建筑数据无法保存/加载：
1. 检查Application.persistentDataPath权限
2. 确认JSON序列化正常工作

## 5. 性能与优化建议

### 5.1 性能监控
1. **建筑数量**: 监控主岛建筑数量，避免过多影响性能
2. **更新频率**: 农田生长检查使用协程而非每帧更新
3. **碰撞检测**: 优化建造模式下的碰撞检查频率

### 5.2 内存管理
1. **精灵资源**: 使用对象池管理常用建筑精灵
2. **预制体缓存**: 动态创建的预制体应缓存复用
