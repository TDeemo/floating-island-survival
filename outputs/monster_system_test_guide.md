# 怪物系统测试指南

## 概述
本文档提供了漂浮大陆游戏怪物系统的测试指南，包括如何设置测试场景、验证AI行为、战斗交互和掉落系统。

## 测试环境要求
- Unity 2022.3 或更高版本
- 项目已导入所有怪物系统脚本
- 玩家预制体 (Player.prefab)
- 武器掉落系统预制体 (需要添加到场景)

## 快速测试步骤

### 1. 生成怪物预制体
在Unity编辑器中：
1. 点击菜单栏 **Tools/创建怪物预制体**
2. 检查控制台输出，确认预制体生成成功
3. 在 `Assets/Prefabs/Monsters/` 目录下查看生成的预制体：
   - `Goblin_Melee.prefab` - 近战怪物
   - `Bat_Ranged.prefab` - 远程怪物

### 2. 设置测试场景
#### 方法A：使用现有场景
1. 打开 `Assets/Scenes/IslandTest.unity`
2. 确保场景包含：
   - 玩家角色（已有Player标签）
   - 地面和障碍物（用于测试避障）
   - 基础光照

#### 方法B：创建专用测试场景
1. 新建场景 `MonsterTest.unity`
2. 添加以下对象：
   ```csharp
   // 1. 玩家生成点
   GameObject playerSpawn = new GameObject("PlayerSpawn");
   
   // 2. 怪物生成区域
   GameObject monsterSpawnArea = new GameObject("MonsterSpawnArea");
   
   // 3. 地面 (Quad 或 Tilemap)
   GameObject ground = GameObject.CreatePrimitive(PrimitiveType.Quad);
   ground.transform.localScale = new Vector3(20, 20, 1);
   
   // 4. 障碍物 (用于测试避障)
   GameObject obstacle = GameObject.CreatePrimitive(PrimitiveType.Cube);
   obstacle.transform.position = new Vector3(3, 3, 0);
   obstacle.transform.localScale = new Vector3(2, 2, 1);
   ```

### 3. 添加测试脚本
1. 创建空对象 `MonsterTestController`
2. 添加组件 `MonsterSystemTest`
3. 配置脚本参数：
   - `Player Prefab`: 拖入 Player.prefab
   - `Monster Prefabs`: 拖入 Goblin_Melee 和 Bat_Ranged
   - 连接UI元素（如果使用UI测试）

### 4. 测试UI设置（可选）
创建Canvas并添加以下UI元素：
- 状态文本 (Test Status Text)
- 玩家生命值文本 (Player Health Text)
- 怪物计数文本 (Monster Count Text)
- 测试按钮 (Test Button)
- 生成怪物按钮 (Spawn More Button)
- 伤害玩家按钮 (Damage Player Button)
- 治疗玩家按钮 (Heal Player Button)

## 测试用例

### 1. AI行为测试
| 测试项 | 预期结果 | 验证方法 |
|--------|----------|----------|
| 巡逻状态 | 怪物在指定范围内随机移动 | 观察怪物移动轨迹 |
| 索敌检测 | 玩家进入范围后怪物开始追击 | 查看控制台日志和怪物行为 |
| 追击行为 | 怪物持续向玩家移动，避开障碍物 | 观察路径是否绕过障碍物 |
| 攻击触发 | 进入攻击范围后播放攻击动画 | 检查动画状态和攻击事件 |
| 状态切换 | 各状态间平滑过渡 | 观察动画和移动无卡顿 |

### 2. 战斗交互测试
| 测试项 | 预期结果 | 验证方法 |
|--------|----------|----------|
| 玩家攻击怪物 | 怪物生命值减少，播放受击动画 | 检查生命值变化和动画 |
| 怪物攻击玩家 | 玩家生命值减少，生命值条更新 | 检查UI显示和生命值 |
| 击退效果 | 受击时向反方向轻微击退 | 观察位置移动 |
| 无敌时间 | 短时间内连续受击不会扣血 | 快速多次攻击验证 |
| 死亡处理 | 生命值归零时播放死亡动画并销毁 | 观察动画和对象销毁 |

### 3. 掉落系统测试
| 测试项 | 预期结果 | 验证方法 |
|--------|----------|----------|
| 普通怪物掉落 | 10%概率掉落木材/石材/木武器 | 击杀多个怪物统计掉落率 |
| 掉落物生成 | 掉落物出现在怪物死亡位置附近 | 观察生成位置 |
| 武器掉落集成 | 武器预制体正确实例化 | 检查场景中武器对象 |
| 资源收集 | 掉落物可被玩家收集 | 玩家靠近掉落物验证 |

### 4. 性能测试
| 测试项 | 预期结果 | 验证方法 |
|--------|----------|----------|
| 多怪物同时活跃 | 帧率保持稳定 (≥30FPS) | 生成10+怪物观察性能 |
| 长时间运行 | 无内存泄漏或性能下降 | 运行10分钟监控资源 |
| 状态机效率 | CPU占用合理 | 性能分析器检查Update开销 |

## 详细测试流程

### 第一阶段：基础功能验证
1. **怪物生成**
   - 运行场景，确认怪物正常生成
   - 检查怪物组件完整性（SpriteRenderer, Collider, Rigidbody2D）
   
2. **巡逻行为**
   - 观察怪物在无玩家时的移动模式
   - 验证随机移动和待机状态切换
   - 检查避障功能（在障碍物旁观察路径）

3. **索敌检测**
   - 控制玩家进入怪物索敌范围
   - 查看控制台输出："Player entered detect range"
   - 观察怪物状态从巡逻切换到追击

### 第二阶段：战斗系统验证
1. **攻击交互**
   - 让玩家进入怪物攻击范围
   - 确认攻击动画播放
   - 检查伤害计算正确性
   
2. **生命值系统**
   - 对怪物造成伤害，验证生命值扣减
   - 对玩家造成伤害，验证生命值UI更新
   - 测试死亡条件触发

3. **掉落系统**
   - 击杀怪物，观察是否有掉落物生成
   - 验证掉落概率符合配置（约10%）
   - 检查掉落物类型（武器/资源）

### 第三阶段：集成测试
1. **多怪物场景**
   - 同时生成多种怪物类型
   - 测试群体AI行为
   - 验证性能表现

2. **完整游戏循环**
   - 玩家探索 → 遭遇怪物 → 战斗 → 收集掉落物
   - 验证各系统协同工作

## 预期输出

### 成功指标
1. **AI行为**：怪物能正确巡逻、追击、攻击
2. **战斗系统**：伤害计算准确，生命值系统稳定
3. **掉落系统**：概率正确，物品生成正常
4. **性能**：帧率稳定，无明显卡顿

### 失败处理
| 问题现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 怪物不移动 | 缺少Rigidbody2D组件 | 检查预制体组件完整性 |
| 不追击玩家 | 索敌触发器未设置 | 检查CircleCollider2D的IsTrigger |
| 无伤害 | 攻击动画事件未绑定 | 验证动画事件调用OnAttackHit |
| 无掉落 | WeaponDropSystem未初始化 | 确保场景中有WeaponDropSystem对象 |

## 调试工具

### 1. 控制台日志
关键日志信息：
- "Starting random move to..." - 巡逻状态开始
- "Player entered detect range" - 索敌触发
- "Setting walking animation" - 追击状态
- "Enemy OnAttackHit called" - 攻击命中
- "Damage: X" - 伤害数值
- "Dropped weapon: X" - 武器掉落

### 2. Gizmos显示
在Scene视图中：
- 红色圆圈：攻击范围
- 黄色圆圈：攻击距离
- 蓝色圆圈：索敌范围
- 怪物移动方向：蓝色射线

### 3. 性能分析器
使用Unity Profiler监控：
- CPU: MonoBehaviour.Update开销
- Memory: 对象实例化与销毁
- Rendering: Draw Calls和批处理
